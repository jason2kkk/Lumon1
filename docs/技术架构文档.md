## 2. 需求挖掘流程

从用户输入到产出需求卡片的完整链路，是系统最核心的功能。

```mermaid
flowchart TD
    START([用户输入场景/关键词]) --> PLAN

    subgraph phase1 [阶段1 搜索规划]
        PLAN[Claude 生成搜索策略<br/>SEARCH_PLANNING_PROMPT]
        PLAN --> OUT1[ ]
    end

    OUT1 --> COLLECT

    subgraph phase2 [阶段2 数据采集]
        COLLECT{并行采集}
        COLLECT --> WEB[Tavily / GPT / Claude<br/>发现 Reddit 帖子 URL]
        COLLECT --> RDT2[rdt-cli<br/>关键词搜索 Reddit]
        COLLECT --> HN2[Algolia API<br/>搜索 HackerNews]
        WEB --> MERGE[合并去重]
        RDT2 --> MERGE
        HN2 --> MERGE
    end

    MERGE --> ENRICH

    subgraph phase3 [阶段3 评论充实]
        ENRICH[rdt read_post 并发<br/>补充高赞帖评论]
    end

    ENRICH --> FILTER

    subgraph phase4 [阶段4 质量筛选]
        FILTER[Claude LLM 判断相关性<br/>POST_FILTER_PROMPT<br/>输出 keep/remove 索引]
    end

    FILTER --> CLUSTER

    subgraph phase5 [阶段5 需求聚类]
        CLUSTER[Claude LLM 按痛点聚类<br/>CLUSTERING_PROMPT<br/>MECE 原则]
        CLUSTER --> REPAIR[修复截断 JSON]
        REPAIR --> FALLBACK[轻量聚类降级]
        CLUSTER --> CARDS
        REPAIR --> CARDS
        FALLBACK --> CARDS
    end

    CARDS([需求卡片列表]) --> CACHE[(缓存 fetched_needs.json)]
```

**典型耗时**：搜索规划 ~30s → 数据采集 ~180s → 评论充实 ~10s → 质量筛选 ~100s → 聚类 ~120s ≈ **总计 7-9 分钟**

## 4. 多角色讨论系统

四个角色共享同一份 `debate_log`，采用话题分段式讨论，每轮 PM-杠精 对抗后导演小结。

```mermaid
sequenceDiagram
    participant U as 用户
    participant D as 导演
    participant PM as 产品经理
    participant C as 杠精
    participant I as 投资人

    U->>D: 点击"开始讨论"
    Note over D: 开场白 + 拆分讨论话题

    Note over I: 后台并行分析<br/>投资视角风险评估

    loop 逐话题讨论
        D->>PM: 话题提问
        PM->>C: 分析观点
        C->>PM: 质疑挑战
        PM->>C: 反驳回应
        C->>PM: 跟进追问
        D->>D: 话题小结
    end

    I->>D: 投资人综合意见
    D->>U: 最终总结判定
```

| 角色 | 定位 | LLM 调用 |
|------|------|---------|
| 导演 | 主持人，分话题、做小结 | 开场 + 每话题小结 + 收尾 |
| 产品经理 | 需求拥护者，推销机会 | 初始分析 + 每话题回应 |
| 杠精 | 真实用户视角，平衡质疑 | 每话题挑战 + 跟进 |
| 投资人 | 商业可行性评估 | 后台 1 次分析 + 收尾意见 |

## 6. POC 产品准入评价

基于「红毛丹准则」三维度评审产品创意是否值得立项。

```mermaid
flowchart LR
    OPP[报告机会点<br/>正则提取] --> FORM[预填表单<br/>名称/用户/痛点/产品方案]
    FORM --> AI[LLM 评估<br/>POC_EVAL_PROMPT]
    AI --> RESULT([清晰用户 / 真实需求 / 简单产品<br/>三维度判定 + 建议])
    RESULT --> PERSIST[(持久化到<br/>poc_evaluations/)]
```

## 7. LLM 调用与角色路由

```mermaid
flowchart TD
    subgraph role_routing [角色路由]
        ROLE_CFG{role_models.json}
        ROLE_CFG --> CC
        ROLE_CFG --> CG
    end

    CC[call_claude / call_claude_stream]
    CG[call_gpt / call_gpt_stream]

    CC --> API_C[Claude API 中转站]
    CG --> API_G[GPT API 中转站]

    CC --> STATS[Token 用量统计]
    CG --> STATS
```

每个讨论角色可独立配置 Claude 或 GPT，通过设置界面切换。非讨论场景的 LLM 调用通过「全局模型」设置统一控制。所有调用自动累计 `prompt_tokens` / `completion_tokens`。
